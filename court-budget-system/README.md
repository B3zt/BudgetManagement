# ระบบจัดการงบประมาณศาล

ระบบจัดการงบประมาณของสำนักงานศาล ครบทั้ง Input (ผังบัญชี) และ Output (รายงาน/แบบฟอร์ม) ตามที่กำหนด สร้างด้วย Next.js 14 (App Router) + Prisma + PostgreSQL + TypeScript + Tailwind CSS

## ฟีเจอร์ที่ทำเสร็จแล้ว

### Input
**1. จัดการผังบัญชี (`/accounts`)**
- เพิ่ม / แก้ไข / ลบ รายการผังบัญชี ทีละรายการผ่านฟอร์ม
- 4 ฟิลด์หลัก: รหัสบัญชี, ชื่อบัญชีของสำนักงาน, รหัสบัญชี GFMIS, ชื่อบัญชีตามผังบัญชี GFMIS
- ตรวจสอบรูปแบบรหัส (สูงสุด 5 ระดับ เช่น `1.1.1.1.1`), แนะนำรหัสถัดไปอัตโนมัติ, มุมมองต้นไม้ขยาย/ย่อได้พร้อมค้นหา
- ผูกรายการระดับ 1 กับ 4 หมวดงบประมาณ

**2. บันทึกข้อมูลเบิกจ่าย (`/disbursements`)**
- กรอกงบประมาณที่ได้รับ + ยอดเบิกจ่ายจริง รายบัญชี ต่อเดือน/ปีงบประมาณ
- กรองตามหมวด, ค้นหา, บันทึกทีละแถวหรือทั้งหมดพร้อมกัน

### Output
**3. รายงานผลการเบิกจ่าย 4 หมวด (`/reports/disbursement`)**
- เลือกรายเดือนหรือทั้งปีงบประมาณ สรุปงบ/เบิกจ่ายจริง/คงเหลือ/% แยก 4 หมวด พร้อมตารางรายบัญชีขยายดูได้
- พิมพ์ / บันทึกเป็น PDF ได้

**4. แบบแจ้งโอนเงินงบประมาณ (`/transfers`)**
- บันทึกแบบแจ้งโอนเงินระหว่างบัญชี (เลขที่หนังสือ, วันที่, บัญชีต้นทาง/ปลายทาง, จำนวนเงิน, ผู้ขอโอน/ผู้อนุมัติ)
- เลือกบัญชีจากผังบัญชีที่มีอยู่ผ่านช่องค้นหาอัตโนมัติ
- แต่ละรายการมีหน้าเอกสารทางการแยก (`/transfers/[id]`) พร้อมพิมพ์ / บันทึกเป็น PDF

**5. รายงานการใช้จ่ายเงินงบประมาณ รายเดือน/รายไตรมาส (`/reports/periodic`)**
- สลับมุมมองรายเดือน (เรียงตามปีงบประมาณ ต.ค.–ก.ย.) หรือรายไตรมาส (Q1 ต.ค.-ธ.ค. ... Q4 ก.ค.-ก.ย.)
- ตารางสรุปยอดเบิกจ่ายแยกตาม 4 หมวด พร้อมยอดรวมงบประมาณ/เบิกจ่าย/% ต่อช่วงเวลา และยอดรวมทั้งปี
- พิมพ์ / บันทึกเป็น PDF ได้

ทุกหน้ารายงานซ่อนเมนูด้านข้างและปุ่มควบคุมอัตโนมัติเมื่อสั่งพิมพ์ (ใช้ browser print → บันทึกเป็น PDF ได้โดยไม่ต้องติดตั้งไลบรารีเพิ่ม)


## การติดตั้ง

ต้องมี Node.js 18+ และ PostgreSQL (ติดตั้งในเครื่อง หรือใช้บริการ เช่น Neon / Supabase / Railway ก็ได้)

```bash
# 1. ติดตั้ง dependency
npm install

# 2. ตั้งค่าฐานข้อมูล
cp .env.example .env
# แก้ไข DATABASE_URL ใน .env ให้ตรงกับฐานข้อมูลของคุณ

# 3. สร้างตารางในฐานข้อมูล
npm run prisma:migrate

# 4. (ไม่บังคับ) ใส่ข้อมูลตัวอย่าง 4 หมวดงบประมาณ
npm run seed

# 5. รันเซิร์ฟเวอร์พัฒนา
npm run dev
```

เปิด http://localhost:3000 จะพาไปที่หน้า `/accounts` โดยอัตโนมัติ

## Deploy บน Vercel พร้อมฐานข้อมูลของ Vercel (Prisma Postgres)

Vercel ให้บริการฐานข้อมูล PostgreSQL ในชื่อ **Prisma Postgres** ผ่านแท็บ Storage ในโปรเจกต์ ขั้นตอนมีดังนี้

1. **Push โค้ดขึ้น GitHub** แล้วไปที่ [vercel.com/new](https://vercel.com/new) เลือก repo นี้เพื่อสร้างโปรเจกต์ (ยังไม่ต้อง Deploy ก็ได้ถ้าฐานข้อมูลยังไม่พร้อม)

2. **สร้างฐานข้อมูล**: ในหน้าโปรเจกต์บน Vercel → แท็บ **Storage** → **Create Database** → เลือก **Prisma Postgres** → เลือก Region (แนะนำ Singapore ถ้าผู้ใช้งานอยู่ไทย) และแพ็กเกจ → ตั้งชื่อ → **Create**

3. **เชื่อมฐานข้อมูลกับโปรเจกต์**: กด **Connect** แล้วเลือกโปรเจกต์นี้ Vercel จะตั้งค่า environment variable `DATABASE_URL` (และ `DIRECT_URL` ถ้ามี) ให้อัตโนมัติทุก environment (Production/Preview/Development)

4. **ดึงค่า env มารันไมเกรชันจากเครื่องตัวเอง** (ต้องมี [Vercel CLI](https://vercel.com/docs/cli): `npm i -g vercel`)
   ```bash
   vercel link          # เชื่อมโฟลเดอร์นี้กับโปรเจกต์บน Vercel
   vercel env pull .env # ดึง DATABASE_URL / DIRECT_URL จริงมาไว้ในเครื่อง
   npx prisma migrate deploy   # สร้างตารางบนฐานข้อมูลจริงบน Vercel
   npm run seed                # (ไม่บังคับ) ใส่ข้อมูลตัวอย่าง 4 หมวดงบประมาณ
   ```

5. **Deploy**: `vercel --prod` หรือ push ขึ้น branch หลักบน GitHub ให้ Vercel deploy อัตโนมัติ

   `package.json` มีสคริปต์ `postinstall: prisma generate` เตรียมไว้แล้ว ทำให้ Vercel สร้าง Prisma Client ให้อัตโนมัติทุกครั้งที่ build โดยไม่ต้องตั้งค่าเพิ่ม

6. เปิด URL ที่ Vercel ให้มา ระบบจะพาไปหน้า `/accounts` เหมือนตอนรันในเครื่อง

**หมายเหตุ:** ทุกครั้งที่แก้ `prisma/schema.prisma` แล้ว deploy เวอร์ชันใหม่ ต้องรัน `vercel env pull .env && npx prisma migrate deploy` อีกครั้งก่อน (หรือหลัง) deploy เพื่ออัปเดตโครงสร้างตารางบนฐานข้อมูลจริงด้วย — Vercel จะไม่รันไมเกรชันให้อัตโนมัติระหว่าง build

## คำสั่งอื่นที่มีประโยชน์

- `npm run prisma:studio` — เปิดหน้าต่างดู/แก้ไขข้อมูลในฐานข้อมูลโดยตรง
- `npx prisma migrate dev --name <ชื่อ>` — เมื่อแก้ไข schema.prisma แล้วต้องการสร้าง migration ใหม่

## โครงสร้างโปรเจกต์

```
prisma/schema.prisma              โมเดลฐานข้อมูล: ChartOfAccount, Disbursement, BudgetTransfer
src/app/accounts/                 หน้าจัดการผังบัญชี
src/app/disbursements/            หน้าบันทึกข้อมูลเบิกจ่าย
src/app/reports/disbursement/     รายงานผลเบิกจ่าย 4 หมวด (พิมพ์ PDF ได้)
src/app/reports/periodic/         รายงานการใช้จ่ายรายเดือน/รายไตรมาส (พิมพ์ PDF ได้)
src/app/transfers/                รายการ + สร้างแบบแจ้งโอนเงินงบประมาณ
src/app/transfers/[id]/           เอกสารแบบแจ้งโอนเงินฉบับพิมพ์ได้ รายฉบับ
src/app/api/accounts/             REST API ผังบัญชี
src/app/api/disbursements/        REST API เบิกจ่าย (GET query fiscalYear/month, POST upsert)
src/app/api/transfers/            REST API แบบแจ้งโอนเงิน
src/components/                   AccountTree, AccountForm, AccountPicker, SidebarNav
src/lib/accountUtils.ts           ตรวจสอบ/คำนวณรหัสบัญชี, จัดเรียง, สร้างต้นไม้, หาหมวดงบของบัญชี
src/lib/thai.ts                   ชื่อเดือนไทย, ปีงบประมาณ, ไตรมาสงบประมาณ, จัดรูปแบบตัวเลข/สกุลเงิน
```

## ขั้นตอนถัดไปที่อาจต่อยอดได้

1. ระบบสิทธิ์ผู้ใช้งาน / เข้าสู่ระบบ (ตอนนี้ทุกหน้าเปิดใช้งานได้อิสระ ไม่มีการยืนยันตัวตน)
2. นำเข้าข้อมูลผังบัญชี 354 แถวจาก Excel/CSV เป็นชุดเดียว (ตอนนี้กรอกทีละรายการตามที่เลือกไว้ตอนแรก)
3. ระบบอนุมัติแบบแจ้งโอนเงิน (workflow หลายขั้นตอน) แทนการบันทึกครั้งเดียว
4. Export รายงานเป็นไฟล์ Excel นอกเหนือจาก PDF
